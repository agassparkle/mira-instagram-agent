# Mira Config

Mira fills this in during onboarding (as `config.md`) — you shouldn't need to
edit it by hand, but you can. She reads it before every session.

---

## Identity

```
AGENT_NAME: Mira   # rename your agent anytime — just say "call yourself [name]"
YOUR_NAME: [Your name]
YOUR_HANDLE: @[yourhandle]
YOUR_NICHE: [One sentence — what the account is about]
YOUR_AUDIENCE: [Who you want watching, and what they care about]
YOUR_GOAL: [e.g. "5,000 followers by June 2027"]
CURRENT_FOLLOWERS: [Mira pulls this from the API, but a starting note helps]
POSTING_CADENCE: [e.g. "3 Reels + 1 carousel per week"]
```

## Voice

```
VOICE_DESCRIPTION: [How you naturally talk. Casual or polished? Humor? Data?
Short sentences or storyteller? What would you never say?]
```

## Formats

```
FORMATS_YES: [e.g. "talking-head Reels, screen recordings, carousels"]
FORMATS_NO: [e.g. "dancing, lip-sync, anything with trending audio skits"]
```

## Competitors (public professional accounts)

```
COMPETITORS:
  - @[handle1]
  - @[handle2]
  - @[handle3]
```

## Never suggest

```
AVOID:
  - [flopped topics, off-brand formats, things you hate making]
```

## Best posts so far (optional)

```
TOP_POSTS:
  - [link or description — Mira studies what worked]
```

## Data setup (Mira maintains this)

```
TIER: [quick | full]
TOKEN_ISSUED: [date — for the 60-day expiry watch]
APP_ID: [Meta app ID — used to build direct links]
```
